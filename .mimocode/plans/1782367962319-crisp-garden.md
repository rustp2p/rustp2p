## Part 2: Deduplicate Config/Tunnel Types

### 2.1 Eliminate main-crate `UdpTunnelConfig` — re-export core's
**File**: `src/config/mod.rs`

Delete the main-crate `UdpTunnelConfig` (lines 296-336) and re-export core's:
```rust
pub use rust_p2p_core::tunnel::config::UdpTunnelConfig;
```

The `From<Config> for core::TunnelConfig` conversion (lines 338-388) already handles the field mapping. Move `default_interface` and `use_v6` into the parent `Config` (they're already there) and set them during conversion.

### 2.2 Eliminate main-crate `TcpTunnelConfig` — re-export core's
**File**: `src/config/mod.rs`

Same approach. Delete the main-crate version (lines 247-281) and re-export core's. The `init_codec` field can default to `LengthPrefixedInitCodec` in the `From` impl.

### 2.3 Simplify `From<Config> for core::TunnelConfig`
**File**: `src/config/mod.rs:338-388`

After re-exporting, the conversion becomes simpler:
```rust
impl From<Config> for core::TunnelConfig {
    fn from(value: Config) -> Self {
        let mut udp = value.udp_tunnel_config.unwrap_or_default();
        udp.default_interface = value.default_interface.clone();
        udp.use_v6 = value.use_v6;
        
        let mut tcp = value.tcp_tunnel_config.unwrap_or_default();
        tcp.default_interface = value.default_interface.clone();
        tcp.use_v6 = value.use_v6;
        
        Self {
            major_socket_count: value.major_socket_count,
            udp_tunnel_config: Some(udp),
            tcp_tunnel_config: Some(tcp),
        }
    }
}
```

### 2.4 Keep `Config` as the user-facing type
The main-crate `Config` remains the single entry point. It contains both tunnel config and protocol config. Users configure everything through `Config` (or `Builder`), and the conversion to `core::TunnelConfig` happens internally.

## Part 3: Processing Logic Optimizations

### 3.1 [HIGH] Add send-side buffer pool
**File**: `src/tunnel/mod.rs:639`

Currently `allocate_send_packet()` creates a fresh `BytesMut` every call. Add a `BufferPool<BytesMut>` for send buffers, similar to the receive side.

```rust
// In TunnelRouter:
send_buffer_pool: Option<BufferPool<BytesMut>>,

// In allocate_send_packet:
pub(crate) fn allocate_send_packet(&self) -> SendPacket {
    let buf = if let Some(pool) = &self.send_buffer_pool {
        pool.get().unwrap_or_else(|| BytesMut::with_capacity(self.send_buffer_size))
    } else {
        BytesMut::with_capacity(self.send_buffer_size)
    };
    SendPacket::with_bytes_mut(buf)
}
```

### 3.2 [HIGH] Zero-copy relay forwarding
**File**: `src/tunnel/mod.rs:363, 371`

Currently `send_to_id` and `send_to_route` copy the entire buffer via `Bytes::copy_from_slice`. Change the signature to accept `Bytes` directly when possible:

```rust
// BEFORE:
pub(crate) async fn send_to_id<B: AsRef<[u8]>>(&self, buf: &NetPacket<B>, peer_id: &NodeID)

// Add a new method for owned data:
pub(crate) async fn send_to_id_owned(&self, buf: Bytes, peer_id: &NodeID) -> io::Result<()> {
    let route = self.route_table.get_route_by_id(peer_id)?;
    self.socket_manager.send_to(buf, &route.route_key()).await?;
    Ok(())
}
```

In the relay path (handle() where `self_id != dest_id`), convert the `Data` buffer to `Bytes` once via `.freeze()` and pass ownership.

### 3.3 [HIGH] Optimize `oldest_route()` with a min-heap
**File**: `rustp2p-core/src/route/route_table.rs:334-355`

Replace the full table scan with a `BinaryHeap` or `BTreeMap<Instant, (PeerID, RouteKey)>`:

```rust
pub struct RouteTable<PeerID> {
    route_table: DashMap<PeerID, (RouteMetric, Vec<(Route, AtomicU64)>)>,
    route_key_table: DashMap<RouteKey, PeerID>,
    // NEW: min-heap for idle detection
    route_times: Mutex<BTreeMap<Instant, (PeerID, RouteKey)>>,
}
```

Update `route_times` when routes are added/removed. `oldest_route()` becomes O(log N) instead of O(N*M).
